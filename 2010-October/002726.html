On 10/07/2010 10:34 AM, Marcus Schäfer wrote:
> Hi Robert,
> 
>>> Attached is a patch that is intended to be a proposal. I'd like to add a
>>> new feature to Kiwi such that Kiwi can create a secondary disk (or
>>> multiple of those) file when building the system image.
>>>
>>> Basically it looks like this:
>>>
>>> <datadisk format="qcow2|raw|vmdk" size="" unit="">
>>>   <datapartition filesystem="" mountpoint="" size="" unit="M|G"/>
>>> </datadisk>
> 
> Hmm, I'm not sure if I understand it correctly but is it true that you
> would like to create another disk image file with some partition
> setup and filesystem ? 

Yes.

> if so I don't understand why you want to let
> kiwi do that ? 

For convenience and kiwi has all information to tie the separate disk
into .xenconfig, .vmx etc.

> I mean that data disk is an add-on not a requirement
> for the system image to work.

Strictly speaking yes. However, a number of applications want separate
disk images to store their data and thus a secondary disk is "required"
for proper function of an appliance.

The creation of a second disk image is a relative frequent request
during Studio demos. I thought I had filed a FATE entry for this in
Studio (which may be the better place for this functionality) but cannot
find it at this time.

> 
> I think this also belongs more to the storage management of the
> system where we unfortunately don't have good tools for. The entire
> management stack suffers imho.
> 
> I'd like to keep building and managing of systems as separate as
> possible. For example when we decided to create the machine configuration
> file too, we ended with the agreement that we create it only to the point
> which allows the machine to run but anything else we leave up to
> the managing tools for that VM... I still have a somewhat bad feeling
> with that but from a user experience it is important to create at least
> the minimum to run the machine, so it was an acceptable compromise.
> 
> Back to the add-on data disk I think that's not urgently required
> to run the machine, is it ?

Nope, it is mostly a convenience feature and I am OK if you would rather
keep these type of things out of kiwi.
