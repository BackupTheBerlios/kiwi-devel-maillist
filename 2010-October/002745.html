the rng must be committed it's the one actually used by the code
just do a git commit and then rebase. As long as you are in your
branch you can change your commits as you like before sending it
out. That works with

   git rebase -i master

you can then interactively change/melt/reword the commits you
have done in your branch. So a commit in your branch is not
problematic and you need a clean state to do a rebase
