> I am wondering if kiwi is supposed to validate ist input xml files?

Kiwi does validate the XML file for syntactic correctness, i.e. matching
tags, proper placement of elements, etc. There is no validation at the
moment for conditionals, such as "no filesystem attribute specified when
required". This part of the validation is on my plate to implement.


>  
> According to my experience a syntactically invalid config.xml file
> results in a runtime failure without a proper error message.

If you install "jing" you will get better error reporting. Kiwi looks
for the "jing" executable and uses it for error reporting. If jing is
not found you will just get a generic parser error. The enhanced error
reporting with jing is version dependent, Kiwi 4.41 is the first version
with this feature. I don't think this was back-ported to previous releases.
