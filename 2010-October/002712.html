On 09/25/2010 07:06 AM, Robert Schweikert wrote:
> Hi,
> 
> Attached is a patch that is intended to be a proposal. I'd like to add a
> new feature to Kiwi such that Kiwi can create a secondary disk (or
> multiple of those) file when building the system image.
> 
> Basically it looks like this:
> 
> <datadisk format="qcow2|raw|vmdk" size="" unit="">
>   <datapartition filesystem="" mountpoint="" size="" unit="M|G"/>
> </datadisk>
> 
> There are pros and cons to re-using the existing <partition> element.
> Although, I would lean more toward adding the "filesystem" and "unit"
> attributes to the existing <partition> element and re-using it as child
> of the <datadisk> element, I've implemented a separate <datapartition>
> for this proposal.
> 
> Having a separate new <datapartition> element should eliminate any
> potential confusion w.r.t. the use of the element. On the other hand
> re-using a modified <partition> element probably lets us piggy-back on
> some of the existing code and eliminates the creation of a new element.
> 
> Thoughts, comments on the feature and proposed schema?

Haven't seen any comments on this, thought I'd follow up.
