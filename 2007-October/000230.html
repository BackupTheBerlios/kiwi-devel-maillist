Hi Folks,

v1.78 is building. This one is a bugfix release to 1.77. 
Thanks for reporting. Watch out the fixed bugs list:

Bugzilla:
=========
    - activated shell set -x mode (#335573)
    - fixed createLiveCD script due to renaming of image (#337391
    - use Recommends flag for packagemanager (#300981)
    - disable use of zypper --catalog option it doesn't exist in newer
      versions. zypper v0.9.0-7 (#335116)
    - fixed users and group matching (#336884)
    - fixed integration of template files while starting to create
      a new root system (#335268) 

I have tried to use zypper for creating images but had some
problems mainly because zypper installs lot more stuff than smart
which I can't figure out why... my guess is that zypper itself
causes a lot dependencies to be included. It's definitly not a
small solution ;)

Another problem is that zypper evolves rapidly (like kiwi) and
so there was an option called --catalog which seems to be replaced
by an option called --repo. The --catalog could be used with a
comma seperated list of catalogs/repo-aliases to use and I assumed
that --repo could do the same but it behaves differently which forces
me to remove this repo selection completely. I hope that the zypper
options for specifying an alternative cache/data dir will be available
for SLE as well so I can implement that in a clean way like it was
done for smart.

I already asked Jan Kupek and hope to get an answer soon.
If so I think version 1.79 will follow rather soon :)

Thanks
