> > I want to have a read-only disk image for a kiosk, where all system
> > writes are to RAM.  I'm trying to decide if I should start with a
> > CD(live) or USB  image description.  Any suggestions?
> 
> I think a liveCD is the best choice here because it operates
> on a read-only media and writes to RAM like you want. The USB
> implementation will use the USB stick as disk and also writes
> to it so maybe not what you want
> 
> Anyway the USB flash disk would be the more cool solution :-)
> It's faster than a CD, smaller and you can store data persistantly
>

I definitely want to take advantage of the speed of a disk, actually Compact Flash card, and the flexibility to make system updates (patches) over time.  But I definitely want the system to be read-only; this will guarantee image consistency over time and the users do not need any persistent storage.

liveCD has all the isolinux boot magic, whereas the CF card needs to be partitioned, with grub installed, etc.  Also, only parts of the system need to be mapped into RAM (/tmp, /var, parts of /etc, parts of /home/user, etc...).  I'm not sure the liveCD is a good start.

Does anybody else out there think this would be a useful "template"?  I would love to see a diskful, read-only image description as a part of Kiwi.
