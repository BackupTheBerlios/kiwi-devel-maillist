On 06/08/2010 02:10 PM, Praveen_Paladugu at Dell.com wrote:
> The kiwi document (openSUSE - KIWI Image System Cookbook) says that lvm support is available for some types of images.
> Adding lvm="true" to the <type> element, should create logical volumes instead of partitions in the final image.
> 
> But the final xml thus created fails schema validation test. Following is the type line I used:
> 	<type boot="oemboot/suse-xxx" lvm="true" filesystem="ext3" bootloader="grub">oem</type>
> 
> Interestingly the "KIWISchema.rng" doesn't seem to have any mention of lvm in it. So, I would expect the Schema validation to fail.
> 
> If kiwi already supports lvm, fixing the KIWISchema.rng should take care of this issue.

Support for LVM was added with commit
c1c4116beb3ad539216cf4a2d1b24ddc10e64efa and is available in Kiwi
versions > 3.76

If you want to use LVM you need to upgrade to an appropriate version of
Kiwi.
