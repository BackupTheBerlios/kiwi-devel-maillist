Hi,

The attached patch creates an early exit if the user does not allow kiwi
to create the root directory and no BaseRoot is specified in the XML
description.

Only implemented for the build procedure at the moment. Probably need
some more refactoring for the create procedure to get an early exit
condition.

IMHO this new message is a bit cleaner than letting the system fail with
the current messages:

Couldn't create root directory: No such file directory
Couldn't create root object

The current messages are more indicative of a kiwi failure, rather than
a user dictated failure.
