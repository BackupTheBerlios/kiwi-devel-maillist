Hi,

The attached patch reduces the scope of the eval around the config.xml
file parsing. The only operation that may fail with an exception is the
parsing, while accessing the resulting document does not generate
exceptions, thus the eval (exception handling) should only enclose the
code that actually may trigger an exception.
