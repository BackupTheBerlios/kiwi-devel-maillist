Hi,

The attached patch avoids a Perl runtime error message from being
printed if the XML file parsing fails. The message being printed by Perl is

"Empty stream at .....KIWIXML.pm line 149"

The reason for the message is that the assignment inside the eval block
fails if the file parsing does not work. The parser already prints the
reason for the failure, thus the user is not left without information
why the error occurred. However, having the Perl error in the messages
is misleading and the user might think is is a problem with Kiwi, when
there is a problem with config.xml instead.
