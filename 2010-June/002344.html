Hi,

The attached patch changes the way validation errors are reported. If
the xml file was converted via XSL the error will be reported against
the converted file. This makes the error reporting more meaningful as
otherwise validation errors are reported that are due to the use of an
older schema version.

Also wrote a little script to deal with the TAB versus spaces preference
issue, applying the script changed a couple of other lines that had
spaces as indents instead of tabs.
