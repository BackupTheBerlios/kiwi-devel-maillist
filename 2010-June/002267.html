> Anyway, here is a patch for the logging module, checking that the
> specified log server port is numeric in nature, and checking the
> constructor argument value.

I added the patch in parts. I like the numeric check but the
parameter check is not required. if tiny is defined I don't care
for the value. Either it's defined then we want a tiny logger
or it's not defined and we want a full logger.

perl allows to use prototypes for functions but the kiwi code
didn't make use of it. So in a strong typed language I would use
a bool type for this but in perl there is no real bool type.
Because of this I use defined or not defined :-)
