Hi,

> Splashimage option is not available in the upstream grub. It is a
> custom patch added by RHEL/CentOS.

ok

> The splashimage option is used by grub to populate the background of
> the grub menu with the image provided as input. The kernel/initrd
> don't play any role  at this time. 

Thanks, I have implemented support for this and see now the red
enterprise something RHEL 5.4 background. A user can influence that
branding by the line:

  <package name="redhat-logos" bootinclude="true" bootdelete="true"/>

> The graphical boot of CentOSx is handled by rhgb (by adding "rhgb" to the
> kernel options while booting to runlevel 5). This is replaced by Plymouth
> in RHEL6x, which has more configurable options (like a choice of theme
> to select). The Plymouth (in rhel6) has a configuration file, but still
> needs "rhgb" to be added to the kernel options. 

the kernel commandline change is easy to handle here. I have added
the following to the rhel jeos example

  <type image="vmx" ... kernelcmdline="rhgb">
    ...
  </type>

it doesn't make much sense for the plain rhel testing but it's the way
to hand over custom kernel options

will checkin the patches soon
