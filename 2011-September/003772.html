Uhm, that really sounds like a bug. It would be great if you can provide
the kiwi log file '/var/log/boot.kiwi'. You can do this as follows: 

1) boot with the boot option:

   kiwidebug=1

2) when the error happens you now get a shell instead of a reboot
   exception. Within that shell store the log file either on the
   target USB pen disk or on another storage device which might be
   available on that system

   if the pen disk doesn't have a partition table and/or filesystem
   you can create that with fdisk, mkfs.ext2 while in the debugging
   shell.

3) stop the machine and grab the stored log file from the pen disk
   and provide it via mail to the list.

That would be great thanks
