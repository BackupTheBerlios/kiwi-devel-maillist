Hi,

> Kiwi sets the type instead of setting individual flags for partitions, which is not supported in the vanilla parted package.
> 
> Kiwi runs "parted set 2 type 0x83" instead of "parted set 2 lvm on". Since this feature is not supported by parted (in RHEL/CentOS), the partitions' properties are not setup properly while setting up the Logical Volumes  for VMX images.
> 
> The feature to setup the partition types seems more like a suse Specific feature. Is it alright to move the "type" setting commands to those of "flag"  settings?
> 
> Settings the flags is well supported in SUSE and RHEL/CentOS based distributions. This would help with future maintenance across distributions.

We can do that. I suggest to add a map which converts the hex ID's
into parted flag names. I think a new function which does it would
be ok. The call is imho only required in the createPartedInput()
function

Could you open a bug for this one ? I have other things on my plate
which I have to take care first
