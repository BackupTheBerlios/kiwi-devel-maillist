    - removed special packages repos which where delivered as part of
      the kiwi packages and moved them into the buildservice:

      1) RHEL: (i386,x86_64)
         http://download.opensuse.org/repositories/Virtualization:/Appliances:/rhel-054/
 
      2) SLE10: (i586,x86_64)
         http://download.opensuse.org/repositories/Virtualization:/Appliances:/suse-sle10/

      3) SLE11: (i586,x86_64)
         http://download.opensuse.org/repositories/Virtualization:/Appliances:/suse-sle11/

      4) 11.4: (i586,x86_64)
         http://download.opensuse.org/repositories/Virtualization:/Appliances:/suse-11.4/

      5) SLE11 (s390,s390x,ppc,ppc64), can't be build in public build service
         http://download.suse.de/install/SLP/SLE-11-SP2-SDK-LATEST/s390x/DVD1


commit df745084f63f8c2224942c5b12624d48af7715ce

    - allow installation via the kiwi install image into a ramdisk.
      In order to activate this feature one has to specify the
      following on the kernel commandline: 'ramdisk=1' The size
      of the ramdisk can be set by ramdisk_size=<value-in-byte>
      This fixes bug (bnc #717499)
