Hi,

> Finally getting some kiwi work in ;) . This patch adds a new
> attribute to the <repository> element called "imageinclude", a
> boolean. The code treats the attribute as having a "false" default
> value. When set to true (for using zypper package manager only at
> this point) the specified repository is added to the image being
> created, thus the repo is available to install additional packages
> or upgrade existing packages from within the running image.

nice feature :) I guess if I overwrite the repo data with the
options --set-repo or --ignore-repos + --add-repo the imageinclude
attribute is lost which would be the 'expected' behavior

What about local repos like iso://... do they end up unmodified
as zypper repo in the image ? In that case it wouldn't make much
sense, would it ? Same for local paths smb mounts and any other
repo type referencing the network the image is being built in
I guess we will leave it up to the user if it makes sense to have
that repo as part of the image as well right ?
