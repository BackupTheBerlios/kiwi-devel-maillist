> > RHEL-05.4 boot image's linuxrc file calls the "initialize" function at the very
> start, which checks for the version of parted installed and reboots because it
> doesn't support the "-machine" option.
> > This test is not necessary for all the live images, since most of them will not
> go through any re-partitioning.
> >
> > RHEL 5.x branch doesn't carry a parted package which supports this option.
> So, RHEL 5.x's ISO image doesn't boot up at all.
> >
> > It will be better to move this check to where the parted executable is
> actually used, instead of making it part of the "initialize" function.
> 
> You are right. I apply the check only for boot image types which actually make
> use of a partitioner now
