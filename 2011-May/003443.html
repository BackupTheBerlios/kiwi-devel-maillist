Hi,

The attached patch contains updates to the ImageCreator, here are the 
details

- get unit test working again
- reset the unit test assert conditions to the expected messages from
   original patch introducing the test
- introduce arguments to prepareBootImage
   retrieving those from the CommandLine object is not following our
   functional separation model as the config dir and target dir for the
   initrd cannot be given on the command line and therefore should not
   be stored on the command line argument
   (A similar change needs to be applied to createBootImage)
- fixed the use of uninitialized variable on line 645
- refactor setting of XML parameters into methods for re-use
   (additional opportunities for calling these new methods remain, to be
    addressed in separate update)

Only had the opportunity to run the VMWare image creation test. Next 
will be adding arguments to createBootImage and taking advantage of the 
new XMLOverride functions.
