Changes Report kiwi v4.90 2011-05-22 - 2011-05-27
--------------------------------------------------
- most important changes for last weeks release

-------------------------------
Cleanup
-------------------------------
commit ef1151e0f1768f6544a448f6a3bba8d2a26e8e31
commit 18235e8580dbd0c4d0f97cb0aed8fdbcb378cced
commit 77b07bf99a94de2b7ca5714029662cebaa038810
commit d5adbad3028bbd433618c6c71586c811e1df6998
commit f5894ffba763360e07698e61d3c22bd3f08d21bd

    - eliminate use of global variable: XMLChangeSet
    - don't store DOM tree references in XML changeset
    - don't create a new KIWICommandLine object in KIWICache
    - eliminate use of global variable: OverlayRootTree
    - eliminate use of global variables:
      * BootVMSystem
      * BootVMSize
      * StripImage
      * ProfilesOrig
      * PrebuiltBootImage
      * ISOCheck
      * CheckKernel
      * LVM
      * GrubChainload
      * FatStorage


This was the last outstanding change to eliminate the use
of global variables by main::... access. Finally the code
is now more clean and also doesn't use recursive kiwi
calls which should make debugging and also unit testing
easier

-------------------------------
P1 Bugs
-------------------------------
commit cb55b610f005c591805f79f003c2012b63c15148

    - fix the problem with an improper exit status on test failure.
      We now use prove (part of Perl) to execute the unit tests.
      Prove generates the proper exit status upon test failure.

commit 7d6d6e94cc14b721cde150c50d8b1e6984592875

    - fixed spec file: The functionality provided with the
      kiwi-desc-usbboot package has moved into a new technology
      which is not provided by another package. Therefore the
      wrong provides entry was removed.

commit b1d6e0508e5e89b50a27feca2f4a3c8d59244367

    - enhanced setupConsole to remove the ttyS0 console if no
      ttyS0 device exists (bnc #695948)

commit c68a807378e97e82139090e675c6ab42b2653485

    - fixed setupConsole() if search string appears as comment

commit 7d80be143062da1769aa0b48ea9e255c486380e5

    - use -t squashfs to mount the installation image. This works
      around the problem if libuuid doesn't know the squashfs
      signature

commit e45e05e3b84b7c35e8f459faad7150dd23bbc74f

    - fixed bootloader setup if split image type is used

commit 17220572a86c4b66f3c03e72cd8194f696b1559b

    - s390: fixed disk online code, wait for disk device to appear (bnc #693031)

commit 2553ea92fe13b547c14ffbdbda04c034762fd685

    - fixed boot included archive installation in prepareBootImage
