HI,

While it is great to have a pretty printer fro the xml conversion, it is 
not so great that the pretty printer swallows the comments.

Not sure if this is an xsltproc problem as I found the following:

(There is a small chance that the comments won't even make it to the 
source tree. The XML parser isn't required to pass comments along to the 
application -- in this case, the XSLT processor -- so if you don't see 
them showing up, try using a different XML parser with your XSLT processor.)

here: http://www.xml.com/pub/a/2000/09/13/xslt/index.html

I also tested modifying the pretty printer by changing:

<xsl:copy />

to

<xsl:comment><xsl:value-of select="."/></xsl:comment>

in pretty.xsl on line 20 but that didn't help either.

The info about xsltproc on my machine:
-> xsltproc --version
Using libxml 20701, libxslt 10124 and libexslt 813
xsltproc was compiled against libxml 20701, libxslt 10124 and libexslt 813
libxslt 10124 was compiled against libxml 20701
libexslt 813 was compiled against libxml 20701

Comments, thoughts?
