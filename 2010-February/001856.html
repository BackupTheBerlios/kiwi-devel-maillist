> Due to a recent mishap with the building of a Xen image and the trick at 
> least my brain played on me with xen vs. nex I thought it might be nice to 
> spare others from this potential trap. The attached patch will rename the 
> bootkernel argument from 'nex' to 'xenboot'.

Thanks for the patch. First I have committed the documentation changes
along with some docu changes I made. Thanks again for doing this

* Changing the bootkernel profile name in the oemboot image
  descriptions seems to be required for your needs but also has
  an impact on many other parts of the system. Additionally this
  name should then be changed consistently over all boot image types
  so vmx is also affected as well as ec2 and others. I need to
  be very careful here. So the patch is still not checked in

* Using XSL to change a free format profile name is something
  we have to prevent. I'm sorry but I want to use XSL transformation
  only if the XML format has changed in some way. In this case
  only the name of a profile is changed. It can lead to strange
  results if we unconditionally change attribute contents of type=text
  I will not add that part of the patch.
