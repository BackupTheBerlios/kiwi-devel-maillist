Hi,

> >I saw that we are creating repository alias names called
> >genName_NUMBER. The NUMBER is different from each build even
> >if the repository setup stay the same. Is that by intention ?
> >This has one problem. kiwi uses the repo cache created by
> >zypper which speed up the build if the cache can be used.
> >But if each new build creates a new repo alias zypper never
> >will use the cache
> >
> >Could you check your genName_ code with respect to this ?
> 
> The attached patch will fix this problem. The number in the alias is
> the digest of the string given as the repo name. The digest will
> always be the same for the same given repo, yet it will be different
> for different repos to avoid alias name collisions.
