On 07/25/2011 12:03 PM, Praveen_Paladugu at Dell.com wrote:
> All
>
>
> While building Centos 05.x Livecd with kiwi on OBS, the right kiwi-tools package doesn't get installed in the boot (initrd) image. I am using the isoboot/rhel-05.4 boot configuration with yum as the package manager.
>
>
> The kiwi-tools package that is built along with the core "kiwi" package doesn't have "/lib/mkinitrd/run-init" executable. The kiwi-tools package distributed rhel-repo directory has the run-init executable.
> So, while using kiwi from command line the live ISO boots up fine. But when building the live ISO from OBS, the iso boot fails complaining about missing run-init executable (because the kiwi-tools from the base kiwi package is picked up).
>
>
> What would be a good way to fix this issue?
> Options are:
> i) Add the run-init exec code to the base packages and make the executable also part of kiwi-tools base package.
> ii) Enable yum priorities support for kiwi and make sure the right package is picked up while building the boot image.
> iii) Of course there is the easy fix of adding the required execs to the archive(root) directory of the isoboot/rhel-05.4 boot configuration.
>
>
> Starting from RHEL6 on, there is not going to be any more mkinitrd support in the base distribution. So is the plan to adopt any of the dracut components or keep re-compiling the mkinitrd for RHEL6 as well?
