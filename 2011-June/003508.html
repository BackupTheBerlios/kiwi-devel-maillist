On 06/06/2011 05:22 AM, Marcus Schäfer wrote:
> Hi All,
>
> I'm currently adding templates for openSUSE 12.1 and while
> doing that I found that we have templates for some older
> distributions in the code. So far I think they are not
> maintained very well and I would like to reduce the amount
> of example template descriptions to something which
> we can maintain
>
> * Package kiwi-doc, provides: /usr/share/doc/packages/kiwi/examples
>    This directory contains examples for 11.2, 11.3, 11.4 and 12.1
>    Is anybody using these examples ? I'd like to change this to
>    the last released version. At the moment: '11.4' Any objections ?

I think we should keep the examples for all the "supported" versions and 
then remove them as each release of openSUSE reaches EOL. We have added 
a lot of features since Kiwi was released with 11.2 and therefore I 
think many peolpe using Kiwi on 11.2 will want to use the development 
tree. As our feature addition rate slows, a natural process, we might 
want to re-visit this once 11.3 reaches EOL.

>
> * Next are the amount of examples in for example
>    /usr/share/doc/packages/kiwi/examples/suse-11.4. We have
>    examples for each kiwi image type and we also document them
>    in the kiwi docs. But we also have additional examples like
>    suse-euca-cloud or suse-min-gnome. I'd like to remove them
>    here and let them be part of the wiki where we reference
>    them. Robert this is mainly a question for you ? Any objections ?

I don't think we want to keep the complete code for these examples in 
the wiki. I am actually working on yet another one of these types of 
examples for the OpenNebula infrastructure (which is why other Kiwi work 
has been taking a back seat lately). I think these are useful and it 
would be great if we can keep them as part of the Kiwi git tree.

The usefulness of these examples certainly goes beyond just the use of 
Kiwi. These types of examples certainly focus more on appliances as a 
whole, therefore I definitely understand why we would not want these as 
part of the core Kiwi git tree, at the moment I think we lack a better 
place for these. I am happy to kick some ideas around if we want to move 
these out, but just removing them does not work for me.

>
> * Next question refers to the enterprise products (SLE). At the
>    moment we distinguish between SLES and SLED. I would like to remove
>    SLED as I think it doesn't make sense. System Imaging's target
>    is definitly the server not the desktop. The description itself
>    just differs in terms of repos and packages. Any objections ?

Yes, I have an issue with this. In the ISV world I live in the SLED 
templates are definitely useful. While the ratio of SLES vs SLED is 
quite high, we do have appliances that need to start with SLED. That 
said I think it would be worth a discussion with PM and others to see if 
we cannot come up with a unified version, i.e. have suse-SLE11 instead 
of suse-SLES11 and suse-SLED11.

>
> * Package kiwi-templates, provides JeOS examples for 11.2, 11.3
>    11.4, 12.1 SLE10 and SLE11. I would like to reduce them to
>    SLE10, SLE11 and the two latest released openSUSE versions (11.3, 11.4)
>    at the moment. Any objections ?

IMHO, as stated above we should keep the "supported" distros until they 
reach EOL.


My $0.02
Robert
