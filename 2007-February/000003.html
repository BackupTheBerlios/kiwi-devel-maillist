Hi Guy,

> This is a discussion that would go WAY beyond the scope of the kiwi project.

I think you are right, I got other objections against the format change
of size with almost the same arguments you wrote. 

> Personal, I don't understand why people insist on seeing values stored
> in attributes. There is nothing that should preclude a value from
> being stored at the element level.  All parsers started that way :-)

That's because writing a parser based on the descriptions of Backus Nauer
is easier without attribute handling whereas XML scheme descriptions behave
the other way round ;)

> From what I can see, these are the following elements that are not taking their value from an attribute but the element itself:
>         <type/>
>         <size/>
>         <compressed/>
>         <packagemanager/>
>         <rpm-check-signatures/>
> 
> If we collapse the value of 'size' into an attribute,  maybe we should be consistent and do that for all elements in the schema.

you are right, I will __not__ change it. I changed the scheme to
handle the currently used format:

	<size unit="M">460</size>

--xsd--

	<xs:element name="size">
        <xs:complexType>
            <xs:simpleContent>
                <xs:extension base="xs:integer">
                    <xs:attribute name="unit" type="xs:string" />
                </xs:extension>
            </xs:simpleContent>
        </xs:complexType>
    </xs:element>

So do you agree, introducing the scheme is now a good thing ;) ?
