Here is something that I have been thinking about for a while and while 
we cannot fix all problems or potential problems related to 
permissions/ownership, I think we can alleviate them a bit.

When a user sets up a configuration tree, files and directories created 
in the overlay tree will have the users umask (from the build system). 
When the overlay tree is copied to the unpacked image the permissions 
are preserved. This is most likely not correct.

I am wondering if kiwi should "fix" the permissions for well known 
places. For example things that end up in /etc, /bin, /usr/bin get root 
ownership. If a user is created in config.xml and files are pulled in 
from the overlay tree into the created user's home directory we should 
probably change the ownership such that the created user owns the files.

I am sure there are other scenarios, just throwing out a couple of examples.

I am not alking about fiddling the permission bits, only ownership.

I think the problem today is generally ignored and I doubt that people 
add chown/chgrp commands into config.sh

Thoughts, comments?
