Hi,

- The attached patch adds support for https repositories with username 
and password. Here is an example specification:

<repository type="rpm-md" username="UNAME" password="PASS">
     <source 
path="https://nu.novell.com/repo/$RCE/SLES11-SP1-Updates/sle-11-x86_64"/>
</repository>

Kiwi now writes a credentials file and takes advantage of the zypper 
support for this file.

- This patch also addresses the problem that in the current 
implementation we end up using the build system zypper configuration 
file "zypper.conf" as zypper is called without the "--config" option.

- Patch includes new consistency check to assure that all credentials 
used with any specified "https" repository are the same. Further the 
consistency check verifies that the username and password attributes are 
always used together.

- Updated the documentation to reflect the changes.

Thanks for the help Peter and Marcus in pointing me in the right 
direction with this when I was stuck.
