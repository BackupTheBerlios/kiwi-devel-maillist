Hi,

The attached patch allows the user to control the boot timeout.

I debated for a while whether to only use this for the timeout during
OEM install or also as the timeout value of the image. I decided that
the values should be consistent, thus the new optional argument
"boottimeout" sets the timeout value for an image being built plus the
timeout for the live image used to install an OEM image. I did not see a
reason to create 2 attributes for the two timeout values.
