Yes, that's why I implemented the new data structure in parallel and
left getImageTypeAndAttributes() alone. With the parallel implementation
we can work our way through the code one module at a time an start
ripping out calls to getImageTypeAndAttributes() one at a time.

This will also reduce our exposure to introducing regressions.

only modulesKIWIXML.pm should the method be removed from the code base.
