Here it is. The attached patch introduces the new data structure into
the XML object and adds a number of accessor methods. Also there is a
new consistency check to make sure only one <type image=myName"...>
exists per <preferences> section. In kiwi.pl I added the setBuildInfo
function to populate the information for the new currentBuildProfile and
currentBuildType variables. I did not yet get rid off
getImageTypeAndAttributes in kiwi.pl as the use of the new
setImageType() for the caching mechanism threw me a bit of a curve ball. ;)

Not sure we really want to change the value of something we read from
the XML file. It might be nicer to make a copy of the data and then
change the value. But I have to think about this a bit more and
understand the way caching works better before I fiddle with this.
