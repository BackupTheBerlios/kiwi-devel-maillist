Hi,

The attached patch improves (IMHO) the post dump behavior for OEM images.

Changes:
1.) Fix misnomer of oem-dumphalt, now called oem-bootwait
~ schema version bump to 4.9
~ xslt included
~ no change in functionality from oem-dumphalt

2.) Add oem-reboot-interactive
~ post a message before rebooting the system after image expansion
~ wait for user interaction

3.) Add oem-shutdown
~ power the system off after dump and image expansion are complete

4.) Add oem-shutdown-interactive
~ post a message before system power off
~ wait for user interaction

I think this provides sufficient flexibility to cover most use cases, at
least all of the use cases I have come across thus far. In addition the
dumphalt name was IMHO quite misleading, I thought it would turn the
machine off, not wait for user interaction.
