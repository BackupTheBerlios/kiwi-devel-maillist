Nice, me too :)
Especially because it's an environment with its own counter which allows 
references. But it happens that if you refer to a chapter 15.1, a table 15.1 
and a listing 15.1, you'll end up with 3 "in 15.1 blah blah". So I did some 
further tuning:

I prefer vref (varioref) for the page references. For this to work correctly 
it is recommended to specify the languge when loading varioref. The varioref 
doc says it is recommended to use the babel package with the same language to 
ensure the text is chosen correctly, so I added that.

Varioref also provides some measures to enhance references like "listiung 
15.1, chapter 15.1" if you specify the label correctly. I did so for listing, 
chapter, section and subsection. I'm not sure if it is wise to change it 
globally, so please give me opinion :)
