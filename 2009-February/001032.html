As you may know from attending (or watching) my FOSDEM 09 talk, I work on the 
documentation at the moment.

environment for code listings named "lstlisting".

This provides labels, captions and avoids the double-dash problem at the same 
time. I tried it and it looks fine to me. OTOH it makes the documentation 
inconsistent if we utilise different techniques here.

I made a patch for kiwi.tex so you can try and compare.
Opinions?
