Hi All,

wish you a happy new year 2008 :-)

version 2.09 was released and we implemented the following
enhancements and bugfixes:

Bugzilla:
=========
  - fixed extractKernel call checks (#350263)

Enhancements:
=============
  - don't turn patterns into a package list if zypper is used.
    request pattern directly with zypper in this case (#350706).
    If you use opensusePattern names in your image description
    together with smart kiwi will load and interpret the pattern
    itself and turn the pattern into a package list. If you use
    zypper this pattern name is now used directly because zypper
    can handle patterns

  - added KIWISatSolver module and use it if libsatsolver is
    available (#348734). The satsolver is a new library for very
    fast dependency solving. kiwi provides a perl module which
    makes use of the satsolver as a plugin. This means if the library
    is there it is used for pattern solving. If not the generic code
    in KIWIPattern is used. As soon as we provide the solvable data
    with the repositories the satsolving is much faster than what
    I do in KIWIPattern

  - added simple html schema documentation. Find it in
    /usr/share/doc/packages/kiwi/kiwi.xsd.html

Other Bugfixes:
===============
  - fixed locale setup and add boot default language if locale is
    specified in system image description
  - added parport and lp kernel modules to boot image descriptions
  - cosmetic change: fixed error messages when stripping down the initrd
  - fixed netboot bootsplash setup, glibc-locale was missing
    when suseGFXBoot was called. this was necessary for 10.3 and 11.0
  - added missing forcearch attribute to instsource
  
Have Fun :-)

Regards,
Marcus
