Hi James,

> I have attached a patch which allows the netboot initrd to use
> ftp/http/https in addition to tftp for file downloads/uploads.  The
> only user-visible changes are that the 'kiwitftpserver' kernel param
> is now 'kiwiserver', and the addition of a 'kiwiservertype' param.
> The 'kiwiserver' param basically does the same thing as before --
> allows you to choose which server to download from.  The
> 'kiwiservertype' param tells what protocol to use.  Valid values here
> are 'tftp', 'ftp', 'http', and 'https'.  Of course, the default is
> tftp.

Great I will give it a test with my PXE setup here. After that I
will check in the stuff :)

> It is a rather large patch, and I am sure I probably broke something
> somewhere.  Everything I have tried so far works, though.  Imaging is
> a lot faster over ftp/http.  One thing it doesn't do yet is
> authentication, which will come later.

Sounds awesome !
