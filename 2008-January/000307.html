Thanks to Cyberorg for the hint about using the downloaded DVD instead
of the commercial double sided DVD.

I don't know if that's what did the trick, or other updates, but I've
got an installation of Kiwi-ltsp working now.

Getting this to work required a couple of steps that weren't in the
instructions.

Here's my notes.

...........................................................................
Fresh SuSE install from downloaded DVD.
Fresh kiwi-ltsp install from http://en.opensuse.org/LTSP using one-click.

Edit sysconfig w/ yast2
  modified: DHCP_IFACE, DHCP_RANGE, DHCP_SUBNET, DOMAIN, NAME_SERVERS, 
  SERVER_IP and SUSE_INSTALL_SOURCE.

Test booting ltsp client - gets DHCP address,
  but fails to load kernel via tftp

disable firewall using yast2.

Test booting ltsp client - gets DHCP address
 and loads pxelinux.0 via tftp.

New Error:
Could not find kernel image: boot/linux

cd /srv/tftpboot/boot
cp initrd-netboot*default linux

Boots to a Suse login screen!
Login as clif works.  
Running as Xterminal with login to server.
...........................................................................

The image of my edits and the output from kiwi-ltsp-setup are at:

http://www.hecl.org/~clif/kiwi-ltsp/sysedit.gif
http://www.hecl.org/~clif/kiwi-ltsp/kiwi-out.txt

This isn't the way I want LTSP to run, but it's a start and I can work it
from here.

Thanks to all, and I'll report my results.
