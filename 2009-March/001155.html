...not without coding it :)

An OEM image is a virtual disk this means you got one file and this
file includes a bootloader a partition table with partitions and data
inside the partitions. It's one black box which you can dump _as it is_
on another disk device. On first deployment the OEM boot code is able
to detect the new disk geometry and will adapt the partition table and
filesystems according to the given preferences in the image description

kiwi supports the installation of such an image onto a disk. 
The process which drives the installation just detects the disks
and asks which disk should be used to install (dump) the OEM image
onto. That installation code could be modified to achieve the goals
you have. The code for this is in the oemboot/suse-dump file in the
kiwi git repo

if you create an OEM image with the format="usb" or format="iso"
attribute specified in the type section kiwi will create such an
install image which you can use for testing.
