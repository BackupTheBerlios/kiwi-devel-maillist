Hi,

we spent some time to make the XML specification more obvious.
As part of this change it's unfortunately no longer possible to
automatically convert the old XML into the new style without
guessing too much.

Robert your work conflicts at some points with what others did.
I tried to merge all the effort together but mostly the documentation
parts where hard to merge. Could you take a look at that again
and include your changes ? That would be great and sorry for
any inconvenience

The following information gives you a workflow
how to convert the old XML into the new format:

1) the image type is now an attribute:

   old: <type ...>oem</type>

   new: <type image="oem"/>

2) the sections:

   pxedeploy , split , vmwareconfig , xenconfig , lvmvolumes , size

   are now type specific. You have to move them into the
   type where they should belong to.

   old: <type >split</type>
        <split>
          ...
        </split>

   new: <type image="split">
           <split>
              ...
           </split>
        </type>
   
3) all oem-* elements needs to be part of the new oemconfig
   section which is also type specific:

   old: <preferences>
           <oem-swap>true</oem-swap>
        </preferences>

   new: <type image="oem">
           <oemconfig>
               <oem-swap>true</oem-swap>
           </oemconfig>
        </type>


After you have done this adaptions everything should work
again... last famous words :-). I plan to submit 4.1 at the
end of this week
