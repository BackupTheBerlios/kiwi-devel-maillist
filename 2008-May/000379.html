Hei folks,

I just committed some changes in the scheme description which may actually 
break something for those of you who dared using --create-instsource already.

Which is:
the element <productinfo> moved from <metadata> to the new group 
<productoptions>.

I did this for consitency and logical reasons. Plus I added two new groups in 
the <productoptions> element which are handled differently.
The information provided there will be used to create metafiles that YaST 
requires for installation and it is rather probable that I modify those 
again, so please be cautious.

Once the module works it will be possible to create a base repository from 
scratch given you have all rpms available :)


To put it in a nutshell: for most of you nothing changes, for those playing 
with the instsource stuff, feel free to complain ;)
