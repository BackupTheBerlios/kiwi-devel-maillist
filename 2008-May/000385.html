Hi All,

because I got that question very often I would like to share
the following information with you. If the image doesn't boot
it's hard to provide the necessary log information. At any
time that happens to me the following process has helped me
so hopefully this information is useful to you too

* if the image was distorted by a wild grub configuration
  or some kind of schaefi partitioning code, just call
  the create step again to start from a clean setup

* boot the image with the kernel parameter

      kiwidebug=1

  In case of PXE remote boot this kernel parameter needs to
  be included into the PXE configuration (append line)

* Now when the error occurs the boot process will stop
  and provides a shell to you. In that shell you can do
  the following:

  a) plug in a USB stick and store the file /var/log/boot.kiwi on it
  b) if no USB modules are available (netboot) copy the file
     /var/log/boot.kiwi on the boot server using atftp

* Provide the boot.kiwi file along with a bugreport on
  bugzilla.novell.com or even better ... send a patch :-))
