Hi James,

> The attached patch implements the profiles support that I had sent mail
> about earlier.  It allows you to maintain one image description while
> still allowing for some variation in the set of packages and drivers
> that are included.  For instance, a good use of profiles would be to
> have a 'gnome' and 'kde' profile for a LiveCD.  The patch adds two new
> command line arguments to kiwi: --list-profiles and --add-profile.  The
> first allows you to simply list the profiles available in a given image.
> Each one has a name and description.  The second option allows you to
> specify which profiles should be included when preparing the image.

really a nice patch. I have a adapted the patch a bit just for
cosmetic reasons:

  - create a listProfiles function and use the kiwi log to print
    the data like the listImage function does

  - use @Profiles in prepare code only, I think it doesn't make
    sense to have it in Migrate,InstSource, etc.. as well

The current head code will insert a schemeversion attribute as
well which will require a version to be present in config.xml.
I'd like to start at 1.4 which is the reason why I changed your
version info from 1.5 back to 1.4. With the current scheme only
the xsd file contains this version information and I found it
quite useless without a mandatory version information in the
config.xml file too ;)

so please find attached your patch adapted to the current head 
code. If you give me your approved by James I will check it in :)

Thanks for your great work
