Just to explain the aftp client is used by initrd to download the files from 
the tftp server ;)

The /srv/tftboot is set in /etc/xinet.d/tfpd

Where the default setting is -s /tftpboot and should be /srv/tftpboot

This because /srv normally is a different filesystem/partition while /tftpboot 
is on the root filesystem ;)

my 2 cents.
