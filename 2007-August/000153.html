It would only work for people who build images with the same repository as the 
host.

And I am for one who does not. I think it only adds a level of complexity when 
it has to check what are the local repositories.

It would only be handy to have a default set of repositories that could be 
called from any config.xml (like a include)

So if the repository changes it changes for all build scripts.
