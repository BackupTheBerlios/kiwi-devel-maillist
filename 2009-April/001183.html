> Apr-20 13:22:08 <1> : Adding bootstrap zypper service:
> http:__download.opensuse.org_distribution_11.1_repo_oss                                      
> failed
> Apr-20 13:22:08 <3> : zypper: System management is locked by the
> application with pid 7961 (/usr/sbin/packagekitd).
> Close this application before trying again.
> 
> At the time I was trying to create the image I was not running any other
> software management tasks. Since the problem is intermittent I suspect
> there is a race condition somewhere.

The packagekitd daemon periodically locks the RPM database and when
this happens while kiwi runs the initial (not chrooted) RPM lock
by the package manager will fail.

I think this is not a kiwi fault but the kiwi behavior is a
bad user experience in this situation. Maybe kiwi should try to
lock the RPM database in a very first step.

Any ideas how to lock the RPM database to prevent other
programs from grabbing the database ?
