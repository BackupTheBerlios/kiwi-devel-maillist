Hi,

> I' m wondering howto build 32bit kiwi image on 64bit server?
> If I start the process with "setarch i386 kiwi --prepare ..." it seems
> to work.

I do that very often with:

   linux32 bash
     kiwi --prepare
     kiwi --create

It's important that both steps prepare _and_ create run with the same
arch setup. If they are mixed the behavior is undefined

> Where is the problem starting the image on a 64bit server, I can' t
> see any flag to create a 32bit image:

I don't know what setarch exactly did but it didn't really run the
process in a 32bit env for all processes and subprocesses

> Jan-28 13:05:22 <2> : Pattern or product match failed for arch: x86_64
> Jan-28 13:05:22 <2> :     a) Check if the Pattern or product is written correctly?
> Jan-28 13:05:22 <2> :     b) Check if the arch is provided by the repo(s)?

your repo is a 32bit one but uname -a got x86_64. The repo doesn't provide
64bit patterns and therefore the lookup failed

I guess if you use linux32 as showed above you will not have that
problems
