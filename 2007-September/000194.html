> > 1.  I was considering using isoinstboot as a re-spin installer method
> > from a bootCD to an empty box.  What would be the preferred method now
> > that isoinstboot is removed?  Oemboot?

I have re-implemented the install CD/DVD support. KIWI version
v1.61 will have it within the oemboot boot image. I have tested
this with 10.3 only but I expect it work for the other oemboot
descriptions as well.

To make use of it the following is needed:

  * include an additional image type with the config.xml file
    of your _system_ image description. For example:

       <type filesystem="ext3" format="iso" boot="oemboot/suse-10.3">vmx</type>

    the format="iso" indicates you want to have a install CD iso to be
    created. The vmx type is mandatory in combination with the oemboot
    boot image

  * create the vmx image

    kiwi --prepare /path/to/system/image/description --root /tmp/mySystem
    kiwi --create /tmp/mySystem -d /tmp

  * If you have multiple <types> in your config.xml make sure you
    select the correct one with "--type vmx"

If all works fine kiwi will tell you:

	Created /tmp/....iso to be burned on CD 

use that file and create the install CD. If you boot from the CD
kiwi will install the virtual disk system image from the CD on your
disk. Everything on your disk will be removed ! Take care about that.

After the installation is done the system reboots, make sure the CD
has been removed from the driver in order to allow the system to boot
from the hard disk. At first boot of the hard disk the kiwi oemboot
boot image will re-partition your disk to the full disk size and the
oemboot initrd will be replaced by the distribution default initrd.

After that the oemboot boot image will boot into your newly installed
system.

All this in v1.61 or from svn 

Have Fun :-)
