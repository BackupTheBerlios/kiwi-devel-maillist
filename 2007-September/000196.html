This is interesting because the usbboot boot image doesn't
do any partitioning. The linuxrc script of this boot type loads
some USB modules, try to detect the USB stick device, probes the
filesystem of the stick, resizes filesystem to existing disk
geometry (partition table remains unchanged), creates some files
(fstab,etc...) and finaly boots into the system of the stick
