I don't know if that has been discussed before or if there are specific 
reasons why Kiwi doesn't use a config file to customize the Kiwi behavior, 
but I had the need for one, so I implemented it. The patch is attached.

The config file is named ".kiwirc" and has to be in the home directory. It's 
Perl syntax and you currently can specify two settings: $BasePath points to 
the base directory where Kiwi lives. $SystemBoot points to the path where the 
boot image descriptions are located.

With these options it's for example possible to directly use a Kiwi SVN 
checkout without having to install it or creating symlinks.

I'm not sure it completely works, though. So if somebody wants to have a look 
or has other comments that would be great.
