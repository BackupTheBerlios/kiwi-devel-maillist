Thanks for the patch. Hmm I'm not sure if I want a .kiwirc file.
The intention is to have all kiwi provided image descriptions at a
specific place which is /usr/share/kiwi/images. The descriptions made
by others can be anywhere and are referenced by the --prepare <path>
option.

I think the basic kiwi modules belongs to the project and right now
you can't add custom modules without a change in the basic kiwi.pl
main script. So I think they should have a fixed path which is
/usr/share/kiwi/modules

I can understand that it is easier for you to test from the svn repo
but I don't see an advantage for non developers :) If you don't
mind could you open an enhancement bug with the patch applied.
I'd like to think about it twice.
