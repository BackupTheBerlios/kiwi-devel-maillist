Yum doesn't return an erorr, if one of the packages in the install list is not available. So, if any of the mandatory packages are missing there is way to find out during the build. 

For example if hwinfo, is not available in any repository, yum would just print a message "No Package hwinfo available" and proceed with installing the rest of the packages in the list. 



Currently, the yum version 3.2.28 has this feature implemented. This version of yum would fail as soon as one of the packages in install list is not available and return an Error code. This version is going to be part of RHEL 6 and not part of RHEL 5.x or 4.x branches.


So, for better error checking, either the yum messages  should be parsed to check for "No Package xxxx available" OR after the yum command completes, "rpm -q" command has to run on all the packages in the install list. I personally prefer the second method.
