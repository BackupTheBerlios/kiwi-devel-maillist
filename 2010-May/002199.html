On 05/18/2010 03:43 AM, Marcus Schäfer wrote:
> Hi Robert,
> 
>> Attached is a patch to support targeted package selection for ISO and OEM 
>> images in addition to Xen and VMware (which we already have).
>>
>> This is useful for at least the following use cases.
>>
>> <packages type="iso">
>>     <package name="yast2-live-installer"/>
>> </packages>
>>
>> <packages type="oem">
>>     <packages name="kvm"/>
>> </packages>
> 
> I like the idea but I changed your patch to be more generic.
> It's now possible to have type specific sections for all types
> we support: iso, net, xen, usb, oem, vmx. Along with this change
> I added a new schema version 4.6 and a style sheet which converts
> type="vmware" into type="vmx".

Nice, but "vmx" may be a bit to coarse grained. As "vmx" has many
formats (VMware, i.e. vmdk, qcow2, etc.) a

<packages type="vmx">
   <package name="vmware-tools">
</packages>

Now pulls the vmware tools in even if I build an image intended for qemu.

I am living with the assumption that the following is possible in the
same config file:

<type image="vmx" .... format="qcow2".../>
<type image="vmx" .... format="vmdk".../>

This may not be allowed and therefore my point may be moot.
