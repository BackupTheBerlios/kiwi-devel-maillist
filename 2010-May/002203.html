Hi,

> Nice, but "vmx" may be a bit to coarse grained. As "vmx" has many
> formats (VMware, i.e. vmdk, qcow2, etc.) a
> 
> <packages type="vmx">
>    <package name="vmware-tools">
> </packages>
> 
> Now pulls the vmware tools in even if I build an image intended for qemu.
> 
> I am living with the assumption that the following is possible in the
> same config file:
> 
> <type image="vmx" .... format="qcow2".../>
> <type image="vmx" .... format="vmdk".../>

Hmm, I think you shouldn't distinguish the contents by the format
of a VM file type. If this is required you should use the profile
mechanism to distinguish the images which are basically of the same
type (vmx).
