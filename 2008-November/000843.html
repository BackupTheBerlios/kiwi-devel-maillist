Hi,

> On 2008-11-04 I opened the bug
> https://bugzilla.novell.com/show_bug.cgi?id=441377 , but I haven' t got
> any response.
> 
> The current kiwi build 3.01 needs the zypper command --reposd-dir, but
> the command --reposd-dir is only available with zypper >= 0.9.1, which
> is only available with openSUSE 11.0.
> 
> I backported zypper from openSUSE 11.0 to SLE 10
> https://build.opensuse.org/package/show?package=zypper&project=home%3Amopp%3Azypper 
> 
> <https://build.opensuse.org/package/show?package=zypper&project=home%3Amopp%3Azypper> 
> 
> and installed the zypper version.
> After installation I got the same problem because, zypper gets executed
> in the chroot environment and the chroot environment is setup from the
> source files.
> 
> I' m really wondering how to build images for systems older than
> openSUSE 11.0?

You have two options here:

1) add the zypper repo as <repository> to your image description and
   you will see the newer zypper version inside the chroot too

2) add all packages in the <packages type="bootstrap"> section and 
   remove the <packages type="image"> section. In that case no
   chroot happens at all
