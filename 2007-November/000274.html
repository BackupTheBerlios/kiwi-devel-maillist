>
> I've got a few questions though.  I've used LTSP 3 and 4, and it looks
> like some things are done a bit differently with kiwi-ltsp.  I could use
> some help with the change-over.
>
> 1. Most of our thin clients have required some tweaking in the lts.conf
> file for such things as monitor X_HORZSYNC and X_VERTREFRESH, and
> loading modules.  I've modified my dhcpd.conf to manually assign the
> workstation name to the clients.  Which lts.conf file do I modify with
> my settings? I have these to choose from:
>
> /srv/kiwi-ltsp-nfs/etc/lts.conf

If you use the above, you need to rebuild the chroot squashfs image
once again by running:

# rm /srv/kiwi-ltsp/ltsp-suse-10.3.i686-0.0.2
# mksquashfs /srv/kiwi-ltsp-nfs /srv/kiwi-ltsp/ltsp-suse-10.3.i686-0.0.2

> /usr/share/kiwi/image/ltsp/suse-10.3/root/etc/lts.conf

> /usr/share/kiwi/image/ltsp/suse-10.3/templates/lts.conf

This is the best place to make changes before running kiwi-ltsp-setup -s
>
> Or is there a better way to pass these settings?

Here are some documentations about LTSP
https://help.ubuntu.com/community/UbuntuLTSP
>
>
> 2.  Most of our thin clients use etherboot, not pxe.  I had thought that
> I might be able to simply change the kernel file to be downloaded in
> dhcp.conf to the "/boot/linux" kernel kiwi-ltsp uses, but when I try to
> boot a client, I get this error when I try that:

>

> process.
>
> Any suggestions what I should do to boot our etherboot clients?
>

I have no idea about implementing etherboot with kiwi, Schaefi?

>
> 3.  I have a ton of directories in /srv from every time that I've tried
> to boot a system.  Is it ok to delete all these in a cron job at night,
> and if so, does anyone have a quick script to do this they would be
> inclined to share?

They are number of times you ran kiwi-ltsp-setup, you need to run that
command only once to set up the server. you can safely remove
/srv/iwi-ltsp-nfs.*
They will not be recreated, so you do not need a script to remove them.

You might want to use kiwi-users mailing list for these discussion as
this is a developer's channel.

Hope you get all your clients working well.
