Hi Fruit Lovers :)

v1.94 is building. The following is a list of bugfixes and
enhancements. With the latest release we have again two enhancements :-)
One is the support for iso:///path/to/iso/file sources. The other
one is an integrity check via the md5 hash. When using kiwi many
people change the boot image descriptions in /usr/share/kiwi/*boot/*.
In principal these descriptions are in a "read-to-use" state and
shouldn't be changed because they control how a system boots. With
the md5 hash provides by the packages kiwi can check whether there
are modifications or not. Additionally you can sign your image
descriptions with the --createhash hash option to be able to check
if someone else has changed the image description

I hope I didn't introduce problems with this change, if so please
report your findings. Thanks

Bugzilla:
=========
   - fixed partition table setup for USB sticks. The problem was
     that the sparse file created for the boot partition add 1MB
     more space than calculated before (#342604)
   - added update of user/group permissions (#342219)

Enhancements:
=============
   - added support for iso:///path/to/iso/file (#338090)

   - added option --createhash to create a description md5sum
   - added md5 based integrity check for image descriptions. As
     soon as the file .checksum.md5 exists in the image directory
     it will be used to check the md5 hash for all files listed in
     the md5 file. Kiwi creates by default checksums for all boot
     images

Other Bugfixes:
===============
   - fixed cleanup code, make sure umount happens when not busy
   - fixed removal of base-system path
   - fixed cache dir name for smart to be a unique name
   - added closeKernelConsole function and rename existing function
     to openKernelConsole which is more what it does
   - fixed initial creation of /dev/tty1-4 devices

Have Fun :-)
