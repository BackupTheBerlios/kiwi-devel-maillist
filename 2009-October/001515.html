Frankly said I'd like to avoid custom kernel options because I think
a kernel shouldn't require them. I agree in allowing options for
setting up the framebuffer (like the vga value which currently can
be configured via config.xml) but options which are required due
to hardware specialities is something I'd like to avoid. So far
I did not receive many requests in that area if it grows to an
urgent issue I'm open for a solution though. 

> I will better 
> wait for you to add such an option in KIWI (probably something like the 
> pxedeploy->commandline option?). Or even I can help making KIWI better by 
> developing this option myself and contributing a patch, but unfortunately, I'm 
> not a perl developer.

good patches are always welcome and perl is an easy language :-)
I think it would be a mistake to allow raw data passed on the commandline.
So an option which ends up on the kernel commandline should be
for a specific setting like I already implemented it for the vga
value. If you like you can just follow the implementation from
the vga value in config.xml. It would be very similar to any other
kind of option and will give you a good starting point into the
kiwi source concerning kernel options.

Do not hesitate to ask here on the list if you need help
