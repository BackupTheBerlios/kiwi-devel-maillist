Hi,

> 	Should I have the fbiterm package installed on my host system,
>   when building 

you don't need fbiterm to be installed on your host system but we
can check if the culprit is really fbiterm that way. Just make sure
you have installed this package on your host system:

http:__download.opensuse.org_distribution_11.1_repo_oss
         /suse/x86_64/fbiterm-0.5.20040304-259.8.x86_64.rpm

if you did that change to a text console

  chvt 1
  
now type

  fbiterm -- ls -y
  echo $?

it should print "2" because ls -y is an invalid option for ls

  fbiterm -- ls -l
  echo $?

should print "0" because everything is ok with the ls command now

can you verify that this is the case ? if yes there must
be another problem causing the trouble. If you can make sure
fbiterm is not the problem I'd like to ask you to provide
the file /var/log/boot.kiwi from your image after it has
been deployed and you have logged in the first time
