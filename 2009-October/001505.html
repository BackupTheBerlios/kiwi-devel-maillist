	Hello KIWI developers!

	First, I want to thank you for the great work, that you have done with KIWI. 
I am using it to build an OEM installation DVD for our application. The 
version that I am using is the latest 3.74 version from the 
Virtualization::Appliances repository on openSUSE 11.1.

	During the preparation and testing of the installation disk, I have 
encountered several issues and I have some questions.

	(1) suseConfig() function does not update the HWCLOCK variable in 
/etc/sysconfig/clock. It is empy by default and this causes hwclock to fail at 
boot time and shutdown. Isn't it a good idea to set this variable too, in 
addition to the TIMEZONE variable?

	(2) When configuring the time zone, suseConfig() moves the time zone file 
over /etc/localtime, instead of copying it.

	(3) The resulting OEM installation ISO image asks the user, if he wants to 
overwrite /dev/sda. But pressing "NO" button does not work -- it overwrites 
the disk no matter if the user answers with "YES" or with "NO"... very nasty!

	(4) How can I customize the OEM installation ISO image kernel command line 
and also the installed (on the hard disk) kernel command line. I want to add 
an option "fbcon=rotate:3" to the kernel command line.
