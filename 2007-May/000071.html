Hi,

I've attached a patch which fixes a case where the bootloader is not
installed correctly if you use the KIWI_INITRD option in config.mac.
Basically the KERNEL_LIST variable was empty so no valid menu.lst was
created for grub.  The patch also introduces two new options for
config.mac: KIWI_KERNEL_OPTIONS and KIWI_BOOT_TIMEOUT.  The first allows
you to specify kernel commandline parameters to be used with the local
boot (useful for displaying a splash screen, etc).  The second simply
sets the grub timeout to the specified value.
