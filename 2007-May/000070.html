Hi Fruit Lovers,

kiwi 1.33 is building and there are some new features but in order
to implement it in a clean way I had to change the
<preferences> -> <type> xml handling a bit.

*** please read ***

Up to now it was only possible to setup one image type. With 1.33
it is possible to add multiple <type> information to config.xml
which allows you to specify different output types while calling kiwi
with --create | -c

  * Selecting a type is done with a new option named --type.
  * Only specified types can be selected, what's not included
    to config.xml can't be selected.

A type is specified with its name and attributes. The old type
specification for example: <type>iso:isoboot/suse-10.3,compressed</type>
is no longer supported and should be changed to:

    <type boot="isoboot/suse-10.3" flags="compressed">iso</type>

You can add additional types for example:

    <type boot="vmxboot/suse-10.3" filesystem="ext3">vmx</type>

and select between them by '--type iso' or '--type vmx'. In order
to have a primary type if you don't specify the --type option the
attribute "primary" exists. A type can be the primary type by
setting primary="true". If you don't specify a primary type the first
entry in the list of types is the primary type.

I have updated the documentation according to this change. I hope
you will find it useful. I myself think it would be great to have
somebody for writing documentation because I think what I wrote
is not easy to understand ;)

I'm sorry for this incompatible change of the type setup but it
would cause us more problems if we remain on the weird value scheme
separated by colons or commas. I hope you agree with the new
style.

Other Changes:
--------------
  - kiwi-pxeboot: install files to /srv/tftpboot (Novell Bugzilla #248008)
  - added patch for netboot and xennetboot linuxrc to allow
    alternative config files. The same search order like PXE
    implements is used (Novell Bugzilla #272337). Thanks to
    Uranellus, great job :)
  - print version information in help output
  - added --set-repo and --set-repotype options. This allows a
    temporary overwrite of the first repository entry within the
    config.xml file.

That's it for 1.33. If you see problems with the new scheme do not
hesitate to tell me or send a patch :-))

Have fun
