Hi all,

v1.43 is building. The following is a list of bugfixes and
enhancements:

Bugzilla:
=========
  - re-added resolv.conf to be part of the chroot system. This was the reason
    for the smart error: "Temporary failure in name resolution" (#290379)
  - added searchSwapSpace() function for live media. include swap
    activation of partition or into a file (#289313)

Enhancements:
=============
  - added baseSetupUserPermissions() method in KIWIConfig.sh to change
    the user and group ownership of the files referenced by /etc/passwd
  - make the <size> attribute an optional value. If you don't set
    it in config.xml kiwi will calculate the image size itself

*** System to Image migration support ***

  - added KIWIMigrate.pm module to turn the running system into an
    image description. This is in an experimental stage and was
    designed for suse(rpm) based systems:
    It would be great if you could give it a try on your system.
    The code only checks filesystem and rpm database information
    together with some magic you can create a description and/or
    report of the system data:
    
    kiwi --migrate mySystem --destdir /tmp/migrate --report

    Optional you can exclude directories with

        --exclude ... --exclude ...

    The code will check for the root partition first and only this
    filesystem will be inspected, NFS mounted data or other partitions
    will not be checked. This is a first version of the system to
    image migration module. I would like to enhance this one as much
    as possible so your help and feedback is highly appreciated.
    Thanks a lot and don't forget to:

Have a lot of fun :-)
