Hi,

> > I think this is a great idea, I have something like this in sax2 too ;)
> > Talking about kiwi there is an attribute called "inherit" which allows
> > you to specify for example:
> > 
> > 	<image name="bob" inherit="/path/to/another/image/description">
> > 		...
> > 	</image>
> > 
> > This will cause the <packages> sections to be prepended which means
> > you are inheriting these information. I found that people don't use
> > this feature very often and maybe more important than the packages
> > are other data like you mentioned. So if there are no objections
> > we can change the inherit attribute to a profile attribute and
> > let that attribute require another config.xml file as value instead
> > of a path to a complete image description to inherit from. All the
> > information of this additional profile xml could be used to manipulate
> > the original information. 
> > 
> > What do you think ?
> 
> Hmm, I am not sure why you would want it to be in a separate file.  I
> think it would be easier to manage if all the profile info was in the
> single config.xml.  That way, you can easily see what things are in
> which profile, etc.  Also, I think you might want some sections to
> belong to multiple profiles, so it works out better that way too
> (<packages profile="foo,bar,baz">).

I don't have a problem to include all in one, I just thought it might
make sense to change the existing "inheritance". Feel free to implement
it your way. I'm fine with that solution
