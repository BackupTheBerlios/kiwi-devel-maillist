Hi,

Often when creating an image description, you want to be able to have
some variation available.  For instance, if you were creating an
openSUSE LiveCD, you might want the option of having either a gnome or
kde desktop.  If you have a very space-optimized image, you may want the
ability to choose things like kernel drivers or some other optional
packages.  For these kinds of things, I think it might be nice to have
"profiles" available to enable the optional bits.

The way I was thinking it could be implemented would be a simple
'profile' attribute on selected elements ('drivers' and 'packages') in
config.xml.  It might be a good idea to have a 'profiles' element too
which lists the various profiles that are available (with description,
etc).  Thoughts?

James
