> 1. I have to delete /tmp/sledlive/etc/X11/xorg.conf to get X server running.

your config looks like it was based on the one I have created.
Therefore I assume the patched version of /etc/init.d/boot exists
in your image as well. In this file you will find something like

        # create config file if not yet done
    if [ ! -f /etc/X11/xorg.conf ];then
        /usr/sbin/sax2 -c 0 -a -i &>/dev/null
    fi

So if you don't have a static xorg.conf as part of your image sax2
is called in order to create one. You can try to call sax2 manually
to see if you can determine the cause for the error
