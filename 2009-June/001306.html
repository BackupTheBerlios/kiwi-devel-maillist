Hi,

> One question though, What's the easiest way to setup a test environment where 
> I can test the scripts in ? If you boot form an oem iso image, there are no 
> bash prompts there, and presumably no editors either. (not that I can do much 
> editing in vi) Would trying the PXE boot method be better? Is there an easy 
> way to get to a bash prompt and curl into the initrd that the OEM boot uses ?

I would do it the following way (guessing suse-11.1):

1) create a copy of the oemboot/suse-11.1 image description:

   mkdir /usr/share/kiwi/image/oemboot/suse-11.1-simon
   cp -a /usr/share/kiwi/image/oemboot/suse-11.1/* \
         /usr/share/kiwi/image/oemboot/suse-11.1-simon

2) edit the oemboot/suse-11.1-simon/config.xml and add the missing "test"
   packages: vi, curl, etc...

3) edit the oemboot/suse-dump file and add:

   systemException "*** interrupted ****" "shell"

   at those places in the code where you want to test something

4) build your image and make sure the boot attribute looks like
   the following:

   boot="oemboot/suse-11.1-simon"
