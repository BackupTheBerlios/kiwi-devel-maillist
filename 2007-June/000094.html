Hi James,

> I've attached a patch which adds a "--locales" option which allows you
> to strip any unneeded locales.  You pass in a list of comma-delimited
> locales (en_US, de, fr, es, etc) and it strips everything except the
> ones you specify.  This is great for space-limited images where you
> don't necessarily want Swahili support :)

Great idea, I think we definitely need that. There are image
descriptions which do that manually within images.sh. Your patch is
great but I would like to have it at another place. Some time ago
I have added a KIWIConfig.sh module which provides global (common for
all distributions) and distribution specific functions. I think
it would be better to have a function in KIWIConfig.sh instead of
the --locales option.

I think there will be more and more low level configuration tasks
which we can't do by YaST or another config tool just because of its
requirements or size. If we would implement each task option by option
we will end in a mess :-) That's why I think it would make more
sense to have a low level config module which allows to call a
function.

What do you think ?
