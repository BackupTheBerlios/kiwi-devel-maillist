Hi,
  First the Caveat - I've been doing a bunch of stuff with an
older version of KIWI-LTSP and updated the system with these commands:

zypper ar http://download.opensuse.org/repositories/server:ltsp/openSUSE_10.3 ltsp
zypper ar http://download.opensuse.org/repositories/openSUSE:/Tools/openSUSE_10.3 tools
zypper in kiwi-desc-ltsp

I *think* the old cruft was removed or overwritten, but I won't bet
my life on it.

Glitch 1) The -l1 option in kiwi-ltsp-setup is no longer working.

  The problem is that the 'dest' variable is not being set correctly.
  I'm guessing some layout changes removed the file that was being used
  to assign a legit value to dest.
  
  I kludged it for now by hardcoding dest=ltsp-suse-10.3.i686...
  in kiwi-ltsp-functions.sh

Glitch 2) The ltsp-client.init startup file is not evaluated
  at boot time.  
  
  There is a test for /usr/lib/ltsp/ltsp_config at the beginning
of this script, and since the /usr/lib/ltsp no longer exists, the
script exits immediately.

  I've put some of my special purpose initialization code into
boot.local, and that works.

  Is there a new preferred place for site-specific customization?

  Otherwise, I'm very happy with 2.38.  It's a big improvement from
the December/January releases, and I'm impressed with the number of
features that are now working.
