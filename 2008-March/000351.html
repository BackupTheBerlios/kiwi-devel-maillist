Hi,

> I tried building and deploying the netboot openSUSE 10.3 image.  The target machine gets it's DHCP address for booting and the initrd and linux image load successfully.  The problem comes during the Kiwi boot system screen:
>  
> Loading KIWI Boot-System...
> ---------------------------
> Creating device nodes with udev
> Boot logging started on /dev/tty1(/dev/console) at Thu Mar 13 00:07:44 2008
> -----> Starting boot shell on tty2
> -----> Error logging enabled on /dev/tty3
> -----> Kernel logging enabled on: /dev/tty4
> -----> Failed to setup DHCP network interface !
> -----> rebootException: reboot in 60 sec...
>  
> I don't understand the DHCP network interface error or how to resolve it. Nor do I know what tty3 is for error logging.

tty3 is a console for detailed information on what kiwi does when
booting. You can switch to this console with the key combination

	Alt-F3

on Alt-F2 there is a shell, on Alt-F4 there is the log of what the
kernel tells us and on Alt-F1 you are back on the info terminal
The information on Alt-F3 will help us to find out why dhcpcd
has failed.
