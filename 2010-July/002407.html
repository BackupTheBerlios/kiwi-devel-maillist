Hi Marcus,

Thanks to Alexander Graf's help we were able to figure out why devices
would go missing when using qemu-kvm to test an image.

When using kvm without using the -usb option, as I usually do, the code
path in the kiwi scripts is such that udevPending is not called.
Therefore ther is a race condition resulting at times in message of not
finding the MBR and other times not finding the CD drive.

The attached patch should fix this problem.
